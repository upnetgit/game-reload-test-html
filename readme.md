俄罗斯轮盘双人对战游戏（Web版）
这是一个基于 Flask 框架的网页版俄罗斯轮盘游戏，融合了策略与运气，支持玩家对战（PVP） 和玩家 vs AI（PVE） 两种模式。游戏包含多种道具，AI 具备智能决策能力，所有操作实时同步，界面直观。

🎮 游戏规则
● 两名玩家初始拥有 2~6 点随机生命值（用 ❤️ 表示）。
● 每回合开始前，系统会随机装填 2~8 发子弹（红色实弹 1，黑色空包弹 0），并随机打乱顺序。
● 每回合开始时，双方会随机获得 2~4 个道具（上限 8 个）。
● 玩家在自己的回合内可以：
  ○ 使用道具（点击道具按钮）
  ○ 直接开枪（选择向自己或向对方开枪）
● 开枪结果：
  ○ 红色子弹：对目标造成 1 点伤害（若已使用刀，则伤害翻倍），之后交换回合。
  ○ 黑色子弹：无伤害，当前玩家可继续行动（回合不结束）。
● 当一方生命值归零时，游戏结束，另一方获胜。

🧰 道具列表及效果
编号	名称	效果
1	knife	使下一次红色子弹伤害翻倍（伤害=2），使用后立即生效，开枪后重置。
2	beer	移除当前子弹并查看其颜色，不结束回合（玩家可继续行动）。
3	handcuffs	使对方下一次获得行动权时被跳过。不能连续两回合对同一人使用。
4	glasses	查看当前子弹的颜色（不消耗子弹）。
5	smoke	回复 1 点生命值（上限 6 点）。
6	phone	随机查看一颗子弹的颜色（不消耗子弹）。
7	epinephrine	偷取对方的某个道具并立即使用。使用后有 3 秒倒计时，需选择道具编号；超时则损失 1 点生命。
手机和放大镜的结果在游戏过程中仅显示“使用了手机/放大镜”，详细结果会在结算页面完整显示。

🎯 游戏模式
● 1 - PVP（双人对战）：两名玩家轮流手动操作。
● 2 - PVE（人机对战）：玩家 1 手动操作，玩家 2 由 AI 控制。
AI 决策逻辑基于原控制台版本，包含以下行为：
● 优先使用烟恢复生命（生命值越低，偷取对方烟的概率越高，最高 0.6）。
● 根据随机概率使用放大镜、手铐、手机、啤酒等道具，优先级：放大镜 > 手铐 > 手机 > 啤酒。
● 在红色子弹时，有概率使用刀增加伤害。
● 在黑色子弹时，有概率使用啤酒跳过当前子弹。
● 每次行动前有 1 秒后端延迟，前端轮询间隔 1 秒，使玩家能看清 AI 的每步操作。

🛠️ 技术栈
● 后端：Python 3 + Flask（轻量级 Web 框架）
● 前端：HTML5 + CSS3 + JavaScript（原生，无外部库）
● 配置管理：YAML 文件（可自定义游戏参数）
● 状态存储：内存字典 + Flask Session

🚀 安装与运行
1. 环境要求
● Python 3.7+
● pip
2. 安装依赖
bash
pip install flask pyyaml
3. 启动服务
bash
python app.py
4. 访问游戏
打开浏览器，访问 http://127.0.0.1:5000/，选择模式开始游戏。

⚙️ 配置文件说明
config.yaml 允许您调整游戏参数，无需修改代码：
yaml
game:
  item_pool: [1,2,3,4,5,6,1,2,3,4,5,6,1,2,3,4,5,6,1,2,3,1,2,3,5,5,7,7,7]
  initial_items:
    player1: [3]   # 玩家1初始道具（手铐）
    player2: [6]   # 玩家2初始道具（手机）
  health_range: [2, 6]      # 初始生命值范围
  bullet_range: [2, 8]      # 每次装填子弹数量范围
  items_per_round_range: [2, 4]  # 每轮获得道具数量范围
  max_items: 8               # 道具上限
  max_health: 6              # 生命值上限

📦 代码结构概览
game.py – 游戏核心类 RouletteGame
● 初始化：设置生命、装弹、道具。
● 道具效果函数：use_knife, use_beer, use_handcuffs 等，每个函数记录消息并更新状态。
● 开枪逻辑：shoot() 判断子弹颜色、伤害、手铐效果，切换玩家，自动重装。
● AI 决策：ai_step() 逐步执行 AI 行为，支持优先级、道具偷取、概率决策。
● 历史记录：_record() 在游戏过程中隐藏手机/放大镜细节，结算时显示完整历史。
● 状态获取：get_state() 返回当前状态供前端渲染。
app.py – Flask 路由
● /：首页（模式选择）
● /start：创建新游戏实例
● /game：游戏主页面
● /result：结算页面
● /api/status：获取当前状态
● /api/action：处理玩家操作（使用道具、开枪、肾上腺素偷取等）
● /api/ai_step：执行一步 AI 动作，返回是否继续
● /api/reset_gun_details：重置弹匣详细显示标志
templates/index.html – 游戏界面
● 实时显示玩家生命、道具、弹匣、消息和历史记录。
● 通过 setInterval 轮询状态（2 秒）。
● AI 回合时启动轮询，每次请求 /api/ai_step，间隔 1 秒。
● 肾上腺素模态框（3 秒倒计时选择偷取道具）。
● 自动滚动历史记录到最新消息。
templates/result.html – 结算页面
● 显示获胜者、最终生命值、完整操作历史（包括手机/放大镜的详细结果）。
● 提供“重新开始”和“返回首页”按钮。

🔧 自定义与调试
● 修改 AI 行为：调整 ai_step 中的概率阈值或优先级顺序。
● 调整延迟：修改 game.py 中 ai_step 的 time.sleep(1) 和前端 setTimeout 的数值。
● 增加历史显示条数：在 get_state 中更改 self.history[-10:] 的数字。
● 清空历史：游戏重置时会自动清空历史记录。
● 调试模式：app.py 已开启 debug=True，便于开发。

📝 注意事项
● 游戏状态存储在服务器内存中，重启服务会丢失所有游戏实例。
● 不同浏览器标签页共享同一会话（基于 Flask Session），如需独立游戏请修改 session 机制。
● 肾上腺素超时逻辑：前端倒计时结束后发送 epinephrine_timeout 请求，后端扣除 1 点生命并移除肾上腺素道具。
● 偷取肾上腺素：如果偷取的道具是肾上腺素，则先损失 1 点生命，然后随机偷取对方另一个道具并立即使用。
● 手机和放大镜的详细结果仅在结算页面显示，游戏过程中隐藏以增加悬念。
